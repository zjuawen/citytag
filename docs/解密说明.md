# API 响应解密说明

## 接口信息

- **URL**: `https://citytag.yuminstall.top/api/interface/v2/device3/{uid}`
- **方法**: `POST`
- **请求体**: `{"encryption": "加密的Base64字符串"}`
- **响应格式**: `BaseRest<String>`，其中 `data` 字段是加密的 Base64 字符串

## 响应结构

```json
{
  "code": "00000",
  "msg": "成功",
  "data": "加密的Base64字符串",
  "pageCount": 0,
  "pageIndex": 0,
  "count": "0",
  "authorized": true,
  "isSessionTimeout": false
}
```

## 解密流程

### 1. 获取响应数据

从响应的 `data` 字段获取加密的 Base64 字符串。

### 2. 解密数据

使用 `DESUtil.des3DecodeECB(token, encrypted_data)` 解密：

- **算法**: 3DES (DESede)
- **模式**: ECB
- **填充**: PKCS5Padding
- **编码**: Base64
- **密钥**: 用户的 token

### 3. 解析 JSON

解密后得到 JSON 字符串，解析为对象列表（`TrackReset` 对象数组）。

**解密方法**: `com/hotddos/citytag/utils/DESUtil.java` (第 63-68 行)

```java
public static String des3DecodeECB(String str, String str2) throws Exception {
    SecretKey generateSecret = SecretKeyFactory.getInstance("desede")
        .generateSecret(new DESedeKeySpec(str.getBytes("UTF-8")));
    Cipher cipher = Cipher.getInstance("desede/ECB/PKCS5Padding");
    cipher.init(2, generateSecret);
    return new String(cipher.doFinal(Encodes.decodeBase64(str2)));
}
```

## 注意事项

1. **Token 必须有效**: 解密使用的 token 必须与加密时使用的 token 相同
2. **响应码检查**: 只有 `code == "00000"` 时才进行解密
3. **Base64 编码**: 响应中的 `data` 字段已经是 Base64 编码的字符串
4. **JSON 解析**: 解密后得到的是 JSON 字符串，需要解析为对象

## 错误处理

如果解密失败，可能的原因：
- Token 不正确或已过期
- 加密数据格式错误
- Base64 解码失败
- 密钥长度处理错误

建议在解密时添加异常处理：

```python
try:
    decrypted_data = decrypt_api_response(token, encrypted_data)
except Exception as e:
    print(f"解密失败: {e}")
```
