## 📖 项目简介

这是一个用于查询 CityTag 设备历史轨迹点的 Python 工具。主要功能包括：

- 🔐 **自动登录**：支持用户名密码登录，自动获取并保存 token
- 🔍 **历史轨迹查询**：查询指定设备在指定时间范围内的历史轨迹点
- 📊 **数据去重**：自动去除重复的轨迹点（基于时间戳和坐标）
- 💾 **数据导出**：将查询结果保存为 JSON 文件，支持合并已有数据
- ⚡ **快捷查询**：一键查询最近3天的轨迹数据（使用默认参数）

## 📦 安装步骤

### 1. 安装依赖

```bash
pip install pycryptodome requests
```

或者使用 requirements.txt：

```bash
pip install -r requirements.txt
```

### 2. 配置默认参数（可选）

编辑 `query_tracks.py` 文件，修改以下常量：

```python
# 默认设备序列号
DEFAULT_DEVICE_SN = "201607813254"

# 默认用户名和密码（可选，如果不设置需要每次手动输入）
DEFAULT_USERNAME = "18659179970"
DEFAULT_PASSWORD = "your_password"
```

## 📚 使用方法

### 1. 快捷查询模式

**功能：** 一键查询最近3天的轨迹数据

**使用步骤：**

1. 运行脚本：
   ```bash
   python3 query_tracks.py quick
   ```

2. 如果首次使用，会提示输入用户名和密码登录

3. Token 会自动保存到 `.citytag_token.json` 文件

4. 查询结果会自动保存到 `history_tracks_3days.json`

5. 如果文件已存在，会询问是否合并数据

**特点：**
- ✅ 使用默认设备序列号
- ✅ 自动查询最近3天数据
- ✅ 自动去重
- ✅ 自动保存到 JSON 文件
- ✅ 支持合并已有数据

### 2. 交互式查询模式

运行脚本后选择 `n`，进入交互式查询：

#### 2.1 历史轨迹点查询

选择 `1` - 历史轨迹点查询

**参数说明：**

- **设备序列号**：要查询的设备 SN（可直接回车使用默认值）
- **时间范围**：选择查询的时间范围
  - `1` - 6小时内
  - `2` - 3天内
  - `3` - 7天内
  - `4` - 30天内
  - `5` - 90天内
  - `6` - 自定义时间范围
- **每页大小**：分页查询的每页大小（默认100）

**示例：**

```
请选择查询类型:
1. 历史轨迹点查询
2. 分页查询
请输入选项 (1/2): 1

请输入设备序列号 (直接回车使用默认值 201607813254): 

请选择时间范围:
1. 6小时内
2. 3天内
3. 7天内
4. 30天内
5. 90天内
6. 自定义时间范围
请输入选项 (1-6): 2

请输入每页大小 (直接回车使用默认值 100): 
```

#### 2.2 分页查询

选择 `2` - 分页查询

**参数说明：**

- **设备序列号**：可选，如果提供则查询指定设备
- **页码**：要查询的页码（默认1）
- **每页大小**：每页返回的数据量（默认20）

## 🎯 功能说明

### 1. Token 管理

**自动保存：** 登录成功后，token 会自动保存到 `.citytag_token.json`

**文件格式：**
```json
{
  "token": "your_token_here",
  "uid": 242487,
  "username": "18659179970"
}
```

**Token 有效期：** Token 会一直保存，直到：
- 手动清除（在交互式查询结束时选择清除）
- Token 过期（需要重新登录）

**清除 Token：** 
- 在交互式查询结束时，会询问是否清除 token
- 或手动删除 `.citytag_token.json` 文件

### 2. 数据去重

工具会自动去除重复的轨迹点，去重规则：

- **时间戳**：精确到分钟
- **坐标**：纬度经度保留3位小数

**去重逻辑：**
```
唯一键 = 时间戳（分钟级） + 纬度（3位小数） + 经度（3位小数）
```

### 3. 数据合并

如果输出文件已存在，工具会询问是否合并数据：

- **合并**：将新查询的数据与已有数据合并，并去重
- **覆盖**：不合并，直接覆盖原文件

### 4. 时间范围筛选

**API 请求参数：** 使用字符串格式的时间范围（`YYYY-MM-DD HH:MM:SS`）

**客户端筛选：** 在接收到 API 响应后，会再次筛选时间范围内的轨迹点

**时间戳格式支持：**
- 日期时间字符串：`"2026-03-04 13:18:33"`
- 毫秒时间戳：`1772601513000`
- 秒时间戳：`1772601513`

### 字段说明

**query_info（查询信息）：**
- `query_time`: 查询时间
- `device_sn`: 设备序列号
- `time_range`: 时间范围描述
- `start_time` / `end_time`: 时间戳（毫秒）
- `start_time_str` / `end_time_str`: 时间字符串
- `total_points_before_dedup`: 去重前的轨迹点数
- `total_points_after_dedup`: 去重后的轨迹点数
- `merged_from_existing`: 从已有文件合并的点数

**device_info（设备信息）：**
- `sn`: 设备序列号
- `name`: 设备名称
- `mac`: MAC 地址
- `uid`: 用户ID
- `status`: 设备状态
- `timestamp`: 最后位置时间
- `updatetime`: 最后更新时间

**track_points（轨迹点数组）：**
- `latitude`: 纬度
- `longitude`: 经度
- `timestamp`: 时间戳（字符串格式）
- `accuracy`: 精度
- `batteryLevel`: 电量
- `confidence`: 置信度
- `id`: 轨迹点ID

## ❓ 常见问题

### 1. Token 过期怎么办？

**现象：** 查询时提示 token 过期或认证失败

**解决方法：**
1. 删除 `.citytag_token.json` 文件
2. 重新运行脚本，输入用户名密码登录

### 2. 没有找到符合条件的轨迹点

**可能原因：**
- 设备在指定时间范围内没有产生轨迹数据
- 时间范围设置不正确
- 设备序列号不正确

**解决方法：**
- 尝试查询更大的时间范围（如7天、30天）
- 检查设备序列号是否正确
- 检查设备最后更新时间

### 3. 查询速度慢

**原因：** 分页查询需要多次 API 调用

**优化建议：**
- 使用快捷查询模式（自动处理分页）
- 减小时间范围
- 增加 `page_size` 参数（最大100）

### 4. 数据重复

**原因：** 多次查询同一时间范围的数据

**解决方法：**
- 使用合并功能，工具会自动去重
- 或手动删除旧文件后重新查询

### 5. 时间戳格式问题

**现象：** 时间戳解析失败

**说明：** 
- API 返回的时间戳可能是字符串格式（`"2026-03-04 13:18:33"`）
- 工具会自动识别并转换
- 如果解析失败，会保留该点并打印警告

## ⚠️ 注意事项

1. **Token 安全**
   - `.citytag_token.json` 文件包含敏感信息，请勿提交到版本控制系统
   - 建议添加到 `.gitignore`

2. **API 限制**
   - 注意 API 调用频率，避免过于频繁的请求
   - 分页查询时，`page_size` 最大为 100

3. **时间范围**
   - API 请求使用字符串格式的时间范围（`YYYY-MM-DD HH:MM:SS`）
   - 客户端会再次筛选时间范围内的轨迹点
   - 时间范围外的点会被过滤掉

4. **数据去重**
   - 去重基于时间戳（分钟级）和坐标（3位小数）
   - 同一分钟、同一位置的多个点会被视为重复

5. **文件合并**
   - 合并时会自动去重
   - 建议定期备份 JSON 文件

6. **Python 版本**
   - 必须使用 Python 3.11 或更高版本
   - 使用 `python3.11` 命令运行脚本
